# 📈 株ポートフォリオ管理アプリ

企業名またはティッカーを入力するだけで、保有・監視したい銘柄の株価とファンダメンタルズを
一覧管理できる個人向けWebアプリ。**日本株 / 米国株 / 香港株**に対応。

- ビルド不要（プレーンな HTML / CSS / JS）
- **常時起動なし・電池消費なし**：データ中継は Cloudflare Worker 上で待機
- スマホ・PC両対応（PCは表、スマホはカード表示／PWAでホーム追加可）
- 起動時に保有全銘柄を自動リフレッシュ
- 銘柄リストと手動入力は端末の localStorage に保存（DB・サーバー不要）

---

## 管理項目

| 項目 | 取得 |
|------|------|
| 企業名 / 市場 / 現在株価 | 自動 |
| 最新決算期の 売上 / 税後利益 | 自動（年次） |
| 実績 PER / ROE / EPS / PBR | 自動 |
| 予想 PER | 自動 |
| 次回決算発表日 | 自動 |
| 会社公表の次期予想 売上 / 税後利益 | 自動は<b>アナリスト予想</b>で代替（`予想`タグ）。会社ガイダンスは手動入力可 |
| 予想 ROE | 予想EPS÷BPSで近似。手動入力可 |

> ⚠️ データ提供元（Yahoo Finance）は会社公表ガイダンスを返さないため、「予想売上/利益」は
> アナリスト予想を表示します（`予想`タグ）。決算短信などの会社予想を使いたい場合は、各銘柄の
> ✎ ボタンから手動入力してください（入力値が優先・`手動`タグ）。

## セットアップ

### 1. プロキシ（Cloudflare Worker）を用意する ※初回のみ・5分

データ取得は Yahoo Finance を中継する Worker が担当します。一度デプロイすれば
ネット上で待機し続けるので、**自分の端末では何も起動する必要がありません**。

1. [Cloudflare](https://dash.cloudflare.com/) に無料登録してログイン
2. 左メニュー **Workers & Pages** → **Create application** → **Create Worker**
3. 名前を付けて **Deploy**（中身は後で差し替え）
4. **Edit code** を開き、エディタの中身を全部消して、このリポジトリの
   [`worker.js`](worker.js) の中身を丸ごと貼り付け → **Deploy**
5. 表示される URL（例 `https://kabu-proxy.xxxx.workers.dev`）を控える

動作確認：ブラウザで `https://～.workers.dev/api/quote?symbol=AAPL` を開いてJSONが
返ればOK。

### 2. アプリ（フロント）を用意する

#### 方法A：GitHub Pages（推奨・どこからでも開ける）
1. このフォルダを GitHub リポジトリにpush
2. リポジトリの **Settings → Pages** で `main` ブランチ / `/ (root)` を公開
3. 発行された URL（`https://ユーザー名.github.io/リポジトリ名/`）をスマホ・PCで開く

#### 方法B：端末ローカルで開く
`index.html` をブラウザで直接開くだけでも動作します（Worker はネット上なので取得可）。

### 3. アプリにプロキシURLを設定する
初回は設定画面が自動で開きます。⚙️ から **手順1で控えた Worker のURL** を貼り付けて保存。
（末尾の `/api/...` は不要。`https://～.workers.dev` まででOK）

以降は **アプリを開くだけ** で全銘柄が自動更新されます。

## 使い方

- 上部の検索ボックスに **企業名（例: トヨタ / Toyota）** か **ティッカー** を入力 → 検索 → ＋追加
  - 日本株: `7203.T`／米国株: `AAPL`／香港株: `0700.HK`（`銘柄コード.HK`）
- ⟳ で全銘柄を手動リフレッシュ
- ✎ で会社予想・予想ROE・次回決算日などを手動上書き（任意）
- 🗑 で削除
